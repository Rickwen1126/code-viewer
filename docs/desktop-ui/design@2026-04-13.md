# Desktop UI Design — Code Viewer

Created: 2026-04-13
Last Updated: 2026-04-14
Status: Draft

---

## 1. Design Goal

把目前 mobile-first 的單欄 UI 擴展為 desktop layout，**充分利用橫向空間**讓 tree + file、commit list + diff、tour list + step detail 能同時顯示，不再需要在頁面間來回切換。

Mobile layout 完全不動 — desktop 是在 `>= 1024px` 時切換到新的 shell 結構。

---

## 2. Current Mobile Architecture (as-is)

```
┌──────────────────────────────────┐
│         ConnectionStatus         │
├──────────────────────────────────┤
│                                  │
│        <main> (Routes)           │
│   Full-page, single column       │
│   route-based navigation         │
│                                  │
├──────────────────────────────────┤
│  [Workspaces] [Files] [Git] [Tours]  │
│         Bottom TabBar (56px)     │
└──────────────────────────────────┘
```

**Navigation model**: 每個 tab 是獨立的 page stack。從 `/files` 點檔案 → navigate 到 `/files/path`（整頁替換）。返回靠 swipe-right 手勢或 browser back。

**State management (2026-04-14 重構後)**:
- 所有 per-workspace 狀態以 `workspaceKey`（opaque stable ID）為主鍵，取代舊的 `extensionId`
- State services 已 extract 成獨立 module：`current-file.ts`, `file-scroll.ts`, `last-location.ts`, `selected-workspace.ts`
- `LastLocationTracker` 在每次 navigation 自動記錄語義位置（tab + route + workspace）
- Session restore 優先順序：LastLocation (semantic) > CurrentFile (convenience) > /files

**Key pain point on desktop**: 看完 code 要回 tree 選另一個檔案，得按 back 回去。Git、Tours 同理。

### 3.1 Activity Bar (左側 icon rail)

取代 mobile 的 bottom tab bar。

| 位置 | 內容 | 說明 |
|------|------|------|
| 上方 | Files / Git / Tours icons | 垂直排列，active 時左側 2px blue border (#569cd6) |
| 下方 | Workspace 選擇器 | 顯示 workspace 首字母 avatar，click 開 popover |

**尺寸**: 48px 寬
**背景**: #252526（比 sidebar 的 #1e1e1e 略深，製造層次感）
**Active 指示**: 左側 2px solid #569cd6 + icon 變亮 (#d4d4d4 → #fff)
**Hover**: icon 背景 #2a2d2e

### 3.2 Sidebar Panel

根據 active tab 顯示不同內容：

| Active Tab | Sidebar 內容 | 對應現有 component |
|------------|-------------|-------------------|
| Files | File tree + search + recent + bookmarks | `FileBrowserPage` |
| Git | Branch info + staged/unstaged + commit list | `GitChangesPage` |
| Tours | Tour list → drill into step list | `TourListPage` |

**寬度**: 280px default, 拖拉調整 (min 200px, max 480px)
**Resize handle**: 右邊緣 4px, `cursor: col-resize`
**Collapse**: 雙擊 resize handle 或 `Cmd+B` toggle
**Persistence**: 寬度存 localStorage `code-viewer:sidebar-width`

### 3.3 Main Content Area

顯示 detail/viewer，根據 route：

| Route | Main Content | 空狀態 |
|-------|-------------|--------|
| `/files` | Empty state: "Select a file" | 顯示 workspace 概要 |
| `/files/*` | Code viewer (語法高亮 + 所有現有功能) | — |
| `/git` | Empty state: "Select a commit or file" | — |
| `/git/diff/*` | Diff detail viewer (text diff + media preview) | — |
| `/tours` | Empty state: "Select a tour" | — |
| `/tours/:id` | Tour step viewer (markdown + code snippet) | — |
| `/open/file` | Workspace resolver → redirect to `/files/*` | — |
| `/open/tour` | Workspace resolver → redirect to `/tours/:id` | — |
| `/open/git-diff` | Workspace resolver → redirect to `/git/diff/*` | — |

**Deep link resolvers** (`/open/*`): 由外部連結觸發（CLI、VS Code、分享連結）。Desktop 遇到 `/open/*` 時 sidebar 保持穩定，main content 顯示 workspace resolver UI；若 workspace 已連線則直接 redirect。

## 5. Responsive Breakpoint Strategy

| Viewport | Layout | Navigation |
|----------|--------|------------|
| < 768px | **Mobile** (current) | Bottom tab bar, full-page routes |
| >= 1024px | **Desktop** | Activity bar + sidebar + content |
| 768–1023px | **Compact desktop** | Activity bar + narrower sidebar (200px) + content |

偵測方式：CSS media query + JS `matchMedia` (for conditional rendering)

```css
@media (min-width: 1024px) {
  /* Desktop shell */
}
@media (min-width: 768px) and (max-width: 1023px) {
  /* Compact desktop — narrower sidebar */
}
```

在 React 層面，`TabLayout` component 根據 `useMediaQuery(768)` 決定 render mobile 或 desktop shell。

## 6. Routing Model Change

### Mobile (current, unchanged)
```
/workspaces        → WorkspacesPage (full page)
/files             → FileBrowserPage (full page)
/files/*           → CodeViewerPage (full page, replaces file browser)
/git               → GitChangesPage (full page)
/git/diff/*        → GitDiffDetailPage (full page)
/tours             → TourListPage (full page)
/tours/:id         → TourDetailPage (full page)
/open/file         → OpenFileResolverPage (workspace resolver → redirect)
/open/tour         → OpenTourResolverPage (workspace resolver → redirect)
/open/git-diff     → OpenGitDiffResolverPage (workspace resolver → redirect)
```

### Desktop (new)
```
/files             → Sidebar: FileBrowserPage | Main: EmptyState
/files/*           → Sidebar: FileBrowserPage | Main: CodeViewerPage
/git               → Sidebar: GitChangesPage  | Main: EmptyState
/git/diff/*        → Sidebar: GitChangesPage  | Main: GitDiffDetailPage
/tours             → Sidebar: TourListPage     | Main: EmptyState
/tours/:id         → Sidebar: TourListPage     | Main: TourDetailPage
/open/*            → Sidebar: 保持穩定          | Main: WorkspaceResolver → redirect
/workspaces        → Popover from activity bar (not routed)
```

**核心差異**: Desktop 不 unmount sidebar 內容。 Mobile 的「整頁替換」變成 desktop 的「main panel 替換 + sidebar 留存」。

**Session restore**: Desktop 開啟時利用 `LastLocationTracker` 存的語義位置自動恢復到上次的 tab + route + scroll position。State services 已用 `workspaceKey` 做 scoping，切 workspace 時所有持久化狀態自動跟著切換。

**Implementation**: `DesktopLayout` component 包含兩個 render slot:
```tsx
function DesktopLayout() {
  const activeTab = deriveActiveTab(location.pathname)
  return (
    <div style={{ display: 'flex', height: '100dvh' }}>
      <ActivityBar activeTab={activeTab} />
      <SidebarPanel activeTab={activeTab}>
        {activeTab === 'files' && <FileBrowserSidebar />}
        {activeTab === 'git' && <GitSidebar />}
        {activeTab === 'tours' && <ToursSidebar />}
      </SidebarPanel>
      <main style={{ flex: 1 }}>
        <Routes>
          <Route path="files" element={<FilesEmptyState />} />
          <Route path="files/*" element={<CodeViewerPage />} />
          <Route path="git" element={<GitEmptyState />} />
          <Route path="git/diff/*" element={<GitDiffDetailPage />} />
          <Route path="tours" element={<ToursEmptyState />} />
          <Route path="tours/:tourId" element={<TourDetailPage />} />
          {/* Deep link resolvers render in main content, sidebar stays stable */}
          <Route path="open/file" element={<OpenFileResolverPage />} />
          <Route path="open/tour" element={<OpenTourResolverPage />} />
          <Route path="open/git-diff" element={<OpenGitDiffResolverPage />} />
        </Routes>
      </main>
    </div>
  )
}
```

## 7. Component Adaptation Strategy

現有 components 不需要重寫，只需要 **extract + compose**。

### 7.1 FileBrowserPage → FileBrowserSidebar

現有的 `FileBrowserPage` 包含：search bar + tree + recent + bookmarks。
Desktop 版直接 reuse，但移除 pull-to-refresh（桌面不需要）。
可能需要調整 padding 和 font size（sidebar 空間較窄）。

### 7.2 GitChangesPage → GitSidebar

現有的 `GitChangesPage` 包含：branch info + staged/unstaged + commit list。
Desktop sidebar 版本保留所有內容，但 commit 展開時不顯示 inline diff（diff 移到 main content）。
點擊 commit row → navigate to `/git/diff/*`，在 main content 顯示。

### 7.3 TourListPage → ToursSidebar

現有的 `TourListPage` 包含：tour list。
Desktop 增加：展開 tour → 顯示 step list（目前只在 detail page 才有 step 切換）。
點擊 step → navigate to `/tours/:id?step=N`，main content 顯示 step detail。

### 7.4 可直接 reuse 的現有 State Services

Phase 2 state cleanup 已將 state 邏輯 extract 成獨立 service module，desktop components 可直接 import：

| Service | 用途 | Desktop 利用方式 |
|---------|------|-----------------|
| `selected-workspace.ts` | workspace 選擇持久化 | WorkspacePopover 直接用 |
| `current-file.ts` | 當前檔案 per-workspace | sidebar file tree highlight |
| `file-scroll.ts` | scroll position per-workspace per-file | code viewer scroll restore |
| `last-location.ts` | 語義位置（tab + route） | session restore 時回到正確的 tab + route |

所有 service 都用 `workspaceKey` 做 scoping，切 workspace 時狀態自動隔離。

### 7.5 新增 Components

| Component | 用途 |
|-----------|------|
| `ActivityBar` | 左側 icon rail |
| `SidebarPanel` | 可 resize 的 sidebar container |
| `ResizeHandle` | 拖拉調整 sidebar 寬度 |
| `WorkspacePopover` | Workspace 選擇 popover |
| `SideBySideDiff` | 雙欄 diff view（desktop 專用） |
| `MediaDiffPreview` | Media file before/after 比較（desktop 專用） |
| `EmptyState` | 各 tab 的空狀態提示 |
| `useIsDesktop` | `matchMedia` hook |

## 10. Side-by-Side Diff (Desktop Enhancement)

Desktop 有足夠寬度做雙欄 diff，取代 mobile 的 unified diff。

```
┌────────────────────────┬────────────────────────┐
│ OLD (commit parent)    │ NEW (commit)           │
│────────────────────────│────────────────────────│
│  10 │ import { A }     │  10 │ import { A }     │
│  11 │ import { B }     │  11 │ import { B }     │
│  12 │ import { C }     │  12 │ import { D }     │  ← red bg / green bg
│  13 │                  │  13 │ import { E }     │  ← added line
│  14 │ function foo() { │  14 │ function foo() { │
│  15 │   return bar     │  15 │   return baz     │  ← changed
│     │                  │  16 │   doExtra()      │  ← added
│  16 │ }                │  17 │ }                │
└────────────────────────┴────────────────────────┘
```

- 兩欄同步 scroll
- 左右分別標 old/new line numbers
- 刪除行：左側 red bg，右側空白
- 新增行：左側空白，右側 green bg
- 修改行：兩側都標色，可做 word-level diff highlight
- 可切換回 unified mode（toggle button）

### 10.1 Media Diff Preview

Diff detail 現已支援 media file preview（image/video）。Desktop diff viewer 需處理三種情況：

| 情境 | 顯示方式 |
|------|---------|
| Text file with hunks | Side-by-side text diff（上面的雙欄 layout） |
| Media file (image/video), working tree | Media preview（居中顯示，合理尺寸限制） |
| Added text file, no hunks | Synthetic "all lines added" text view |

Media diff 在 desktop 可以做更好的 before/after 比較：
- **Overlay slider**: 左右拉桿疊合新舊圖片
- **Side-by-side preview**: 新舊圖片並列（利用雙欄空間）
- **Toggle**: 點擊切換 old/new

目前 mobile 只顯示 current version preview，desktop 可以在有足夠空間時顯示 before/after 並列。

## 12. Key Architectural Decisions

### Q1: Sidebar components 要 refactor 還是 reuse？

**建議**: Extract shared logic into custom hooks, render different UI based on `isDesktop`。

例如 `FileBrowserPage` 目前是整頁 component。可以 extract tree + search + recent 邏輯到 `useFileBrowser()` hook，然後：
- Mobile: `FileBrowserPage` render full-page version（現有）
- Desktop: `FileBrowserSidebar` render compact version（只取需要的部分）

或者更簡單：讓現有 component 接受 `variant: 'page' | 'sidebar'` prop，調整 padding/layout。

### Q2: Desktop routing 要改 route 結構嗎？

**建議**: 不改。保持 `/files`, `/files/*`, `/git`, `/git/diff/*` 等現有 routes。
Desktop layout 根據 route prefix 推斷 active tab + sidebar 內容。
只是 render 方式不同：mobile 全頁替換，desktop sidebar + main 並列。

### Q3: View Transitions 在 desktop 怎麼處理？

**建議**: Desktop tab switch 保留 crossfade（150ms）。
子頁面切換（如切換檔案）改為 instant（無 slide 動畫），因為 sidebar 不動、只有 main content 換內容，slide 反而突兀。

### Q4: Sidebar 和 main content 之間怎麼通訊？

**建議**: 繼續用 React Router 的 URL 作為 single source of truth。
Sidebar 點擊 → `navigate('/files/path')`。
Main content 讀 route params render 內容。
不需要額外的 shared state。

### Q5: Desktop 怎麼利用新的 state services？

**建議**: 直接 import。Phase 2 state cleanup 已經把所有 per-workspace 狀態抽成獨立 module，desktop 不需要再重新實作。

- Session restore: `readLastLocation(workspace)` → 取得上次的 tab + route → `navigate()` 到正確位置
- File tree highlight: `readCurrentFile(workspace)` → 取得當前檔案 path
- Scroll restore: `readSavedFileScroll(workspace, path)` → code viewer 自動處理
- Workspace 切換: `selectWorkspace()` 後所有 state 自動切到新 workspace 的 scope

State services 都以 `workspaceKey` 為主鍵，並保留 `rootPath` / `extensionId` fallback migration。Desktop 不需要擔心 key collision。

### Q6: Deep link resolvers 在 desktop 怎麼顯示？

**建議**: `/open/*` routes 在 desktop main content 中渲染 workspace resolver UI，sidebar 保持當前 tab 的內容不動。Resolver 完成後 redirect 到目標 route（如 `/files/path`），此時 sidebar 自動切到對應 tab。

若 workspace 已連線且匹配，resolver 直接 redirect 不顯示 UI（零閃爍）。

## 13. Non-Goals (這次不做)

- **Editor tabs / multi-file open**: 暫時維持單檔案 focus，不做 tab strip
- **Split editor (左右兩個 code viewer)**: 複雜度高，先不做
- **Terminal panel**: 不是 code viewer 的範疇
- **Minimap**: 效能代價高，暫緩
- **Right panel (outline/references)**: 先保持在 popover/modal，之後再考慮固定 panel
