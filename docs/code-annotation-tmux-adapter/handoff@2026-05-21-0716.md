## Why This Exists

The old Code Viewer annotation todo still says:

```text
Extension -> tmux Vault Input -> Codex
Per-workspace tmux pane binding (.codeviewer/panel.json)
```

That is now superseded by the accepted `tmux-adapter` contract:

```text
one code-viewer workspace
  -> one active Codex-capable binding slot
```

Code Viewer should not keep its own `.codeviewer/panel.json` as binding truth,
and it should not target a specific Codex pid, pane, or session as product
identity. The workspace cwd is the durable identity input; `tmux-adapterd`
resolves that into a live binding.

## Non-Negotiable Boundaries

- Do not use or disturb ports `4800` and `4801` outside Code Viewer itself.
- Do not make Code Viewer spawn tmux directly.
- Do not store pane id / pid / Codex session id as Code Viewer source of truth.
- Do not let frontend pass arbitrary cwd. The extension must derive cwd from
  `vscode.workspace.workspaceFolders[0].uri.fsPath`.
- Do not put real secrets or local tokens in config/docs.
